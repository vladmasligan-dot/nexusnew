# Розгортання Nexus на Railway.com

## 📋 Вимоги

- GitHub обліковий запис
- Railway обліковий запис (реєстрація безкоштовна)
- Цей репозиторій на GitHub

## 🎯 Крок за кроком

### 1. Підготовка GitHub репозиторію

```bash
# Ініціалізуйте Git (якщо потрібно)
git init

# Додайте всі файли
git add .

# Фіксуйте зміни
git commit -m "Initial commit: Nexus social network"

# Додайте remote репозиторій та push'те
git remote add origin https://github.com/YOUR_USERNAME/nexus.git
git branch -M main
git push -u origin main
```

### 2. Реєстрація на Railway

1. Перейдіть на https://railway.app
2. Натисніть "Login" або "Get Started"
3. Натисніть "Continue with GitHub"
4. Дозвольте Railway доступ до ваших репозиторіїв

### 3. Створення проекту в Railway

1. На панелі Railroad натисніть "New Project"
2. Виберіть "Deploy from GitHub repo"
3. Знайдіть ваш репозиторій `nexus`
4. Натисніть "Import"
5. Дозвольте Railway доступ до репозиторію

### 4. Додавання PostgreSQL сервісу

1. На сторінці проекту натисніть "+ New"
2. Виберіть "Add from Marketplace"
3. Пошукайте "PostgreSQL"
4. Натисніть "PostgreSQL"
5. Натисніть "Create"

Railroad автоматично створить базу даних та встановить змінну `DATABASE_URL`.

### 5. Налаштування змінних середовища

1. На сторінці проекту перейдіть на вкладку "Variables"
2. Натисніть на сервіс вашого додатку
3. Додайте наступні змінні:

| Змінна | Значення | Примітка |
|--------|----------|---------|
| `JWT_SECRET` | `your-super-secret-key-min-20-chars` | Генеруйте надійний ключ |
| `NODE_ENV` | `production` | Обов'язково |
| `PORT` | `3000` | Railway використовує динамічний PORT |

**Приклад надійного ключа:**
```
aB3$xQ9@mK2#vL8%wP1!rT6^sY4&uI0*
```

### 6. Налаштування Procfile

Файл `Procfile` вже створений:
```
web: node server.js
```

Переконайтесь, що він знаходиться в корені проекту.

### 7. Розгортання

1. Натисніть "Deploy"
2. Чекайте завершення розгортання (зазвичай 2-3 хвилини)
3. Коли розгортання завершиться, натисніть на URL в розділі "Deployments"

### 8. Ініціалізація Бази Даних

Після першого розгортання потрібно ініціалізувати таблиці:

**Опція 1: Через Railway CLI**
```bash
# Встановіть Railway CLI
npm install -g @railway/cli

# Авторизуйтесь
railway login

# Запустіть ініціалізацію
railway run "node -e \"require('./database/init.js')\""
```

**Опція 2: Через веб-інтерфейс**
1. У Railway перейдіть до вкладки "Logs"
2. Натисніть "Run Command"
3. Введіть:
```
node -e "require('./database/init.js')"
```

## ✅ Перевірка розгортання

1. Відкрийте URL вашого додатку
2. Зареєструйтесь з новим обліковим записом
3. Спробуйте додати пост
4. Перевірте чи все працює

## 🔗 Отримання URL вашого сайту

URL вашого сайту можна знайти:
1. На сторінці проекту Railway
2. У розділі "Deployments"
3. Натисніть на найновіше розгортання
4. Скопіюйте URL

Приклад: `https://nexus-production.up.railway.app`

## 🔐 Безпека

1. **Никогда не комітьте .env файл!** ✅ Він вже в `.gitignore`
2. **Змініть JWT_SECRET** на надійний ключ
3. **Використовуйте HTTPS** (Railway автоматично)
4. **Регулярно оновлюйте залежності**

```bash
npm audit
npm update
```

## 🛠️ Управління розгортаннями

### Переглядання логів
1. У Railway натисніть на сервіс
2. Перейдіть на вкладку "Logs"
3. Фільтруйте за часом

### Перезавантаження додатку
1. Натисніть на сервіс
2. Натисніть "Redeploy"

### Откат до попередньої версії
1. Перейдіть на "Deployments"
2. Виберіть попередню версію
3. Натисніть "Redeploy"

## 🚨 Проблеми та рішення

### Проблема: Додаток не запускається

**Рішення:**
1. Перевірте логи в Railway
2. Переконайтесь, що `package.json` правильний
3. Перевірте наявність `Procfile`

### Проблема: Помилка підключення до БД

**Рішення:**
```
1. Переконайтесь, що PostgreSQL сервіс запущений
2. Перевірте DATABASE_URL у Variables
3. Перезавантажте додаток
```

### Проблема: "Module not found"

**Рішення:**
```bash
# Переінстальте залежності на Railway
npm install
```

### Проблема: Білий екран при завантаженні

**Рішення:**
1. Відкрийте браузер Developer Tools (F12)
2. Перевірте Console та Network tabs
3. Скопіюйте помилку та перевірте логи Railway
